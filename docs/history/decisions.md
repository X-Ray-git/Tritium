# 决策日志

## 2026-07-19：固定产品边界

背景：旧工程包含关注、点赞和较复杂的回复入口，与当前阅读客户端定位不符。

决策：主导航仅保留内容与设置；删除远端点赞、关注、回复、收藏、私信、发现和 AI
功能，公开数量保持只读展示。回复采用单层弹层查看。

后果：新增功能必须先确认不突破只读边界，不要恢复已删除的交互入口。

## 2026-07-19：固定品牌与包名

决策：品牌色固定为 RGB(57, 97, 255) / `#3961FF`，包名改为
`io.github.xraygit.tritium`。品牌色不从图标或系统动态生成。

后果：旧包无法被新包覆盖安装；图标需要另行适配，但不能反向改变应用主题色。

## 2026-07-19：参考工程职责分离

决策：Hydrogen 只参考功能模块和知乎接口，Auto Folo 只参考设计、维护架构与发布
流程。Tritium 不复制其源码，也不修改参考工程。

## 2026-07-19：发布必须可追溯

决策：版本发布使用 `scripts/release.sh` 创建版本提交和 annotated tag；GitHub Actions
校验 tag 与 `pubspec.yaml`、执行分析和测试、注入正式签名并发布 APK。普通提交和未
签名构建不创建 GitHub Release。

## 2026-07-19：登录诊断不得泄露凭据

决策：登录检查点仅在 Debug 构建输出安全事件，隐藏查询参数和 Cookie 值；WebView
结束回调缺失时允许进度或 12 秒超时撤下遮罩。

后果：排障日志可以直接用于判断状态机，但仍不得要求用户提供 Cookie 或完整 URL。

## 2026-07-19：与 Auto Folo 共享 Android 签名

决策：Tritium 的 GitHub Actions 复用 Auto Folo 的 Android keystore，通过 Tritium
仓库独立的四项 Secrets 注入；证书和密码不进入源码仓库。

后果：两款应用的签名身份和密钥轮换相互关联，密钥泄露会同时影响两者。共享签名不
改变两个应用的包名、数据目录和更新关系。
